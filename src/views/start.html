GitHub Requests

The GitHub request application is a tool designed to streamline and automate the process of managing and tracking requests for GitHub. This includes the adding, removal or editing of members. Further documentation can be found here.

Before you start:

  • New users must have a GitHub handle.

  • New user has to be associated with an existing team; otherwise, the new team must be created in the same request.

  • Every new team has to have at least one maintainer (Team Maintainer Docs).

  • Non permanent users/members must provide the expected end date of the work.
